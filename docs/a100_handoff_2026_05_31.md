## Prepare Data

Use `N=1500`, `M=150`.

```bash
. reference/code/SpecForge/.venv/bin/activate
python src/prepare_domain_splits.py --train-size 1500 --eval-size 150
```

This writes:

```text
.artifacts/rl_project/data/domain_n1500_m150/
  gsm8k_train.jsonl
  gsm8k_eval.jsonl
  mmlu_train.jsonl
  mmlu_eval.jsonl
  sharegpt_train.jsonl
  sharegpt_eval.jsonl
  all_train.jsonl
  mixed_eval.jsonl
  manifest.json
```

The split script uses `--seed 42` by default. It is deterministic given the same Hugging Face dataset contents/order:

- GSM8K: shuffle `train` with seed 42, then first 1500 train and next 150 eval.
- MMLU: shuffle `auxiliary_train` with seed 42 for train; shuffle `test` with seed 43 for eval.
- ShareGPT: shuffle `train` with seed 42, filter invalid rows, then first 1500 train and next 150 eval.
- `all_train` and `mixed_eval` are shuffled with Python `random.Random(42)`.

For strict reproducibility across machines, keep the generated JSONL files and `manifest.json` from the A100 run with the final artifacts. Upstream HF datasets can theoretically change, so the generated JSONL files are the source of truth after creation.

Fairness rule:

- `gsm8k`, `mmlu`, `sharegpt`: train on 1500 examples each.
- `all`: train on 4500 examples.
- All drafters should use the same number of optimizer steps.

## Full Training Run

Recommended first full run on A100:

```bash
. reference/code/SpecForge/.venv/bin/activate
MAX_STEPS=7500 SAVE_INTERVAL=2500 bash src/run_domain_train_all.sh
```

This sequentially trains:

```text
gsm8k -> mmlu -> sharegpt -> all
```

Outputs:

```text
.artifacts/rl_project/outputs/domain_n1500_m150/
  qwen3-0.6b-eagle3-gsm8k/
  qwen3-0.6b-eagle3-mmlu/
  qwen3-0.6b-eagle3-sharegpt/
  qwen3-0.6b-eagle3-all/
```

If time is tight, use this quick comparable run:

```bash
MAX_STEPS=2500 SAVE_INTERVAL=2500 bash src/run_domain_train_all.sh
```

Do not mix `2500` and `7500` checkpoints in the final matrix unless explicitly reporting it as an ablation.

If A100 training is fast, consider a longer run because the local smoke test improved from step 900 to step 2500. A good schedule is:

```bash
MAX_STEPS=15000 SAVE_INTERVAL=2500 bash src/run_domain_train_all.sh
```

Then evaluate checkpoints at 2500/5000/7500/10000/15000 for one drafter first. If accept length is still rising, continue to 30000. Keep the final comparison fair by using the same checkpoint step for every drafter.

Do not change `--speculative-num-draft-tokens 4` for the main matrix. If you want an ablation on draft-token count, do it after the main fixed-4 result is complete.
